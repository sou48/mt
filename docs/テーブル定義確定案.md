# MT テーブル定義確定案

## 位置づけ

本書は [バックエンド仕様](/home/takeshi/multi-translate/docs/バックエンド仕様.md) を、実装着手できるテーブル定義レベルまで具体化したものです。  
初期バックエンド実装では PostgreSQL を前提にします。

## 共通方針

- すべての主要テーブルで `created_at`, `updated_at`, `deleted_at` を持つ
- 論理削除を採用する
- 主キーは `bigint` 相当を前提にする
- 外部キーは原則として物理整合性を持たせる
- 履歴テーブルは更新前スナップショットを保存する

## 命名規則

- テーブル名は複数形
- 中間テーブルも複数形
- 状態値は英字の固定値
- 日時は `*_at`
- 真偽値は `is_*`

## 1. companies

### 用途

会社マスタ。

### カラム

| カラム名 | 型 | Null | 制約 / 用途 |
| --- | --- | --- | --- |
| id | bigint | NO | PK |
| name | varchar(255) | NO | 会社名 |
| created_at | timestamp | NO | 作成日時 |
| updated_at | timestamp | NO | 更新日時 |
| deleted_at | timestamp | YES | 論理削除 |

### インデックス / 制約

- PK: `id`
- index: `name`
- unique 相当の運用制約:
  - `deleted_at is null` の有効レコード内で完全一致禁止

### 備考

- 類似名は DB 制約ではなくアプリケーション警告で扱う

| --- | --- | --- | --- |

| updated_at | timestamp | NO | 更新日時 |

### インデックス / 制約

- index: `company_id`
- check: `role in ('user', 'admin')`

## 3. projects

### 用途

案件マスタ。

### カラム

| カラム名 | 型 | Null | 制約 / 用途 |
| --- | --- | --- | --- |
| id | bigint | NO | PK |
| company_id | bigint | NO | FK -> companies.id |
| name | varchar(255) | NO | 案件名 |
| is_unclassified | boolean | NO | 未分類仮案件フラグ |
| created_by_user_id | bigint | YES | 作成者 |
| created_at | timestamp | NO | 作成日時 |
| updated_at | timestamp | NO | 更新日時 |
| deleted_at | timestamp | YES | 論理削除 |

### インデックス / 制約

- PK: `id`
- FK: `company_id`
- FK: `created_by_user_id`
- index: `company_id`
- index: `(company_id, name)`
- unique:
  - 同一会社かつ有効レコード内での完全一致案件名禁止
  - `is_unclassified = true` の案件は会社ごとに 1 件

### 備考

- 未分類仮案件は通常一覧に出す
- ただし UI 上で「未分類」を明示する
- 未分類仮案件は名称変更、削除不可

## 5. messages

### 用途

受信、返信、下書きの一元管理。

### カラム

| カラム名 | 型 | Null | 制約 / 用途 |
| --- | --- | --- | --- |
| id | bigint | NO | PK |
| project_id | bigint | NO | FK -> projects.id |
| message_type | varchar(20) | NO | `received` / `reply` / `draft` |
| channel_type | varchar(20) | NO | `email` / `chat` / `phone_memo` / `other` |
| subject | varchar(500) | YES | メール系件名 |
| source_sender_name | varchar(255) | YES | 元送信者名 |
| source_sender_address_or_account | varchar(255) | YES | 元送信元情報 |
| source_sent_at | timestamp | YES | 元メッセージ日時 |
| registered_by_user_id | bigint | YES | 受信登録者 |
| created_by_user_id | bigint | YES | 返信 / 下書きの作成者 |
| source_text | text | YES | 取り込み原文 |
| source_language | varchar(50) | YES | 原文言語 |
| translated_text | text | YES | 翻訳文 |
| translated_language | varchar(50) | YES | 翻訳文言語 |
| japanese_text | text | YES | 日本語文 |
| partner_text | text | YES | 相手言語文 |
| language_pair | varchar(100) | YES | 例: `日本語<>韓国語` |
| signature_id | bigint | YES | FK -> signatures.id |
| signature_snapshot | text | YES | 当時の署名本文 |
| created_at | timestamp | NO | 作成日時 |
| updated_at | timestamp | NO | 更新日時 |
| deleted_at | timestamp | YES | 論理削除 |

### インデックス / 制約

- PK: `id`
- FK: `project_id`
- FK: `registered_by_user_id`
- FK: `created_by_user_id`
- FK: `signature_id`
- index: `project_id`
- index: `(project_id, source_sent_at desc, created_at desc)`
- index: `message_type`
- index: `channel_type`
- check: `message_type in ('received', 'reply', 'draft')`
- check: `channel_type in ('email', 'chat', 'phone_memo', 'other')`

### 運用ルール

- `message_type = 'received'`
  - `registered_by_user_id` を原則必須
- `message_type in ('reply', 'draft')`
  - `created_by_user_id` を原則必須
- 件名は `channel_type = 'email'` のときのみ利用
- 表示並びは `source_sent_at` 優先、なければ `created_at`
- 返信と下書きは `japanese_text` と `partner_text` の両方を正本として扱う

## 8. dictionary_entries

### 用途

システム共通辞書、会社共通辞書のエントリ。

### カラム

| カラム名 | 型 | Null | 制約 / 用途 |
| --- | --- | --- | --- |
| id | bigint | NO | PK |
| scope_type | varchar(20) | NO | `system` / `company` |
| company_id | bigint | YES | 会社共通辞書のときのみ使用 |
| source_term | varchar(255) | NO | 原文 |
| target_term | varchar(255) | NO | 訳語 |
| note | text | YES | メモ |
| language_pair | varchar(100) | NO | 例: `日本語<>韓国語` |
| created_by_user_id | bigint | YES | 作成者 |
| updated_by_user_id | bigint | YES | 更新者 |
| created_at | timestamp | NO | 作成日時 |
| updated_at | timestamp | NO | 更新日時 |
| deleted_at | timestamp | YES | 論理削除 |

### インデックス / 制約

- PK: `id`
- FK: `company_id`
- FK: `created_by_user_id`
- FK: `updated_by_user_id`
- index: `scope_type`
- index: `company_id`
- index: `(scope_type, language_pair, source_term)`
- check: `scope_type in ('system', 'company')`

### 備考

- システム共通辞書は `company_id = null`
- 会社共通辞書は `company_id != null`

## 実装時の注意

- 類似名判定は SQL 制約ではなくアプリケーション層で実装する
- 未分類仮案件は会社作成時に自動作成する想定にする
- `messages` の検索性を考え、必要に応じて全文検索導入を後から検討する
- 添付保存先は初期実装ではローカル固定だが、パス設計は移行しやすくする
