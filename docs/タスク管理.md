# MT タスク管理

## 目的

未実装項目や後続フェーズの作業を、やり忘れなく管理するためのファイルです。優先度と実装フェーズの両方で整理します。

## フェーズ定義

- フェーズ1: 初期バックエンド土台
- フェーズ2: 業務運用に必要な機能拡張
- フェーズ3: 実運用向け強化

## 優先度定義

- `P1`: 最優先。これがないと初期実装が成立しない
- `P2`: 重要。初期リリース後すぐ必要になりやすい
- `P3`: 後続対応。必要だが初期実装の必須ではない

## 仕様確定済みメモ

- ユーザーは社内ユーザー単位
- ユーザーは 1 社所属
- 一般ユーザーは担当案件だけ閲覧
- 管理者は全システム管理者
- メッセージは必ず案件に属する
- 未分類受信は会社ごとの未分類仮案件へ保存
- メッセージ種別は `受信 / 返信 / 下書き`
- 返信は実送信ではなく、MT 内で管理する文面
- 辞書は `システム共通 / 会社共通`
- 署名はユーザー単位
- 添付ファイルは受信、返信の両方で扱う
- 添付実体は初期実装では VPS ローカル保存

## 更新ルール

- 仕様確定時は、本ファイルにも反映する
- 初期実装対象から外した項目は削除せず、後続フェーズへ移す
- 実装時はタスク完了チェックと関連ドキュメント更新をセットで行う
