# MT 技術選定

## 目的

初期バックエンド実装に使う技術を決め、選定理由と却下案を明確にするための文書です。

## 前提

- 現在のフロントエンドは HTML / CSS / JavaScript の静的構成
- 現在のアプリサーバーは Node.js + Express
- 本番の公開入口は Traefik
- 既存の Node 実行基盤がある
- 初期実装では速度よりも安全に段階移行できることを優先する

## 選定結果

### 1. バックエンド言語 / ランタイム

- 採用: Node.js

理由:

- 既存の `server.js` と `package.json` があり、現在のプロジェクトと連続性が高い
- フロントエンドと同じ言語で実装でき、文脈切替が少ない
- 既存デプロイ導線を流用しやすい

### 2. Web フレームワーク

- 採用: Express

理由:

- 現在すでに依存関係に入っている
- 初期 REST API には十分
- モックから段階的に API を増やしやすい

却下:

- Next.js API Routes
  - 現在のフロント構成と不連続
- Fastify
  - 悪くないが、いま優先すべきは新規導入の最小化

### 3. DB

- 採用: PostgreSQL

理由:

- Docker で扱いやすい
- JSONB や検索拡張の余地がある
- 将来的な履歴保存や監査に向いている

### 4. ORM / クエリ層

- 採用: Prisma

理由:

- マイグレーション管理がしやすい
- スキーマ定義が明瞭
- Node / Express と相性が良い

代替候補:

- Knex
  - 柔軟だが、型とスキーマの一貫性でやや弱い
- Drizzle
  - 有力だが、このリポジトリではまだ前提がない

### 5. 認証

- 採用: セッション認証
- 初期方針: サーバー側セッション + Cookie
- 現在の保存先: `express-session` の `MemoryStore`
- 将来候補: PostgreSQL または Redis

理由:

- ブラウザ業務アプリに向いている
- JWT より失効制御が分かりやすい
- セキュリティ運用が素直

### 6. パスワードハッシュ

- 採用: bcrypt

理由:

- 実績がある
- Node で一般的

### 7. バリデーション

- 採用候補: Zod

理由:

- API 入出力のスキーマ化がしやすい
- フロント、バック双方で再利用しやすい

### 8. ファイル保存

- 採用: VPS ローカルディスク
- 保存パス管理: DB に `stored_path` を保存

理由:

- 初期構築が最短
- インフラ依存を増やしすぎない

将来課題:

- S3 互換ストレージへの移行余地

### 9. メール送信

- 採用候補: Nodemailer

用途:

- パスワードリセットメール送信

理由:

- Node で定番
- SMTP ベースで導入しやすい

### 10. 検索

- 初期方針: PostgreSQL の `ILIKE` ベース + 必要な index

理由:

- 初期実装の複雑性を抑えられる
- 別検索基盤を追加せずに始められる

将来課題:

- メッセージ件数増加時の全文検索最適化

### 11. API 仕様管理

- 採用: JSON 契約は Markdown、仕様形式は OpenAPI 3.1

理由:

- 実装の JSON 契約を固定しつつ、将来の機械可読仕様にも接続しやすい

### 12. AI 連携

- 採用: OpenAI / Google Gemini / Anthropic Claude の API 連携
- 実装方針: サーバーサイド中継 + モックフォールバック

理由:

- API キーをブラウザへ露出させない
- セッション設定に応じてプロバイダーを切り替えられる
- 開発時はモックで画面検証を継続できる

### 13. フロントエンド UI

- 採用: HTML / CSS / Vanilla JavaScript
- テーマ: ダーク / ライト切替対応

理由:

- 既存実装との連続性が高い
- 小規模業務アプリとしては依存を増やさず保守しやすい

## 推奨スタックまとめ

- ランタイム: Node.js
- Web: Express
- DB: PostgreSQL
- ORM: Prisma
- 認証: セッション認証
- セッション保存先: `MemoryStore`（暫定）
- ハッシュ: bcrypt
- バリデーション: Zod
- メール送信: Nodemailer
- ファイル保存: VPS ローカルディスク
- AI 連携: OpenAI / Gemini / Claude + モック
- フロント: HTML / CSS / Vanilla JavaScript
- 公開入口: Traefik

## 導入順

1. PostgreSQL
2. Prisma
3. セッション認証
4. bcrypt
5. Zod
6. Nodemailer

## 補足

- 現在のフロントは維持し、API 接続へ段階移行する
- いきなりフルリプレースはしない
- AI / 翻訳 API のサーバー中継は実装済み
- セッション永続化ストアは今後の改善項目
