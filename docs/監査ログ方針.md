# 監査ログ方針

## 目的

本番運用時に、誰がどの重要操作を行ったかを追跡できる状態を維持するための方針です。

## 対象操作

- ログイン、ログアウト、パスワードリセット
- 会社、案件、メッセージ、署名、辞書の作成、更新、削除
- 添付ファイルのアップロード、削除、ダウンロード
- AI 設定変更
- 管理者によるユーザー操作

## 記録単位

- 実行日時
- 実行者ユーザー ID
- 実行者メールアドレス
- 対象リソース種別
- 対象リソース ID
- 操作種別
- 結果
- リクエスト元 IP
- 必要に応じて差分の要約

## 初期実装での扱い

- 業務データの更新前スナップショットは履歴テーブルで保持する
- HTTP アクセス監査はアプリケーションログへ出力し、集約基盤へ転送できる形式を維持する
- 機微情報、パスワード、API キー、添付本文そのものは監査ログへ出さない

## 今後の実装方針

- アプリケーションログは JSON 形式で構造化する
- `requestId` を全リクエストへ付与し、API ログと障害ログを突合できるようにする
- 将来的に `audit_logs` テーブルまたは外部ログ基盤へ集約する
- ダウンロード系は本文ではなくメタデータのみ記録する
