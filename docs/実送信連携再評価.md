# 実送信連携の要否再評価

## 現時点の結論

- フェーズ3終了時点では実送信連携は採用しない

## 理由

- 現在の MT は翻訳、下書き、履歴管理に責務を限定している
- 実送信を持つと送信失敗再送、送信済み監査、送信元なりすまし対策が追加で必要になる
- メール、チャット、外部 SaaS ごとに連携方式が大きく異なる

## 再評価条件

- 運用上、外部メールクライアントとの二重管理コストが大きい
- 顧客から送信一元化要件が明確に出る
- 監査ログ、送達確認、送信元認証まで含めて責任分界を整理できる

## 再評価時の確認事項

- 送信失敗時の再送制御
- 送信済み本文と添付の監査保持
- SPF / DKIM / DMARC を含む送信ドメイン管理
- 各外部サービスの API 制限と障害時フォールバック
