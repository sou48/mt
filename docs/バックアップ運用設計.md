# バックアップ運用設計

## 対象

- PostgreSQL データ
- 添付ファイル実体
- `.env` を除く運用設定ファイル

## 方針

- DB は日次フルバックアップを取得する
- 重要更新が多い期間は 1 時間ごとの差分または WAL 保全を検討する
- 添付ファイル領域は日次でスナップショットまたは rsync 互換方式で退避する
- バックアップ先は本体ホストと別ストレージへ置く

## 保持期間

- 日次: 14 世代
- 週次: 8 世代
- 月次: 6 世代

## 復旧手順の最小要件

- DB 復旧手順を README ではなく運用手順書へ分離する
- 添付ファイルは DB 復旧時点と整合する世代を復元する
- 復旧後に `/api/health`、ログイン、会社一覧、添付ダウンロードを確認する

## 定期確認

- 月 1 回は復元テストを実施する
- 復元テストでは別環境へ DB と添付を戻して整合確認する
- 失敗時は原因と対策を運用記録へ残す
