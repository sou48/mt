# S3 互換ストレージ移行方針

## 現状

- 添付ファイルは VPS ローカル保存
- 保存先は `src/lib/attachments.js` で抽象化し、DB 上の `storedPath` は `local://...` を保持する

## 移行方針

- 保存 API は `save / delete / resolve` の責務で揃える
- 将来 `s3://bucket/key` 形式または内部識別子へ差し替え可能にする
- ダウンロード時はストレージ実装ごとに実体解決する

## S3 移行時に必要な追加項目

- バケット名
- リージョン
- 認証情報
- サーバーサイド暗号化設定
- ライフサイクル設定
- 署名付き URL 方針

## 移行手順案

- 新規保存から S3 へ切り替える
- 既存 `local://` データをバックグラウンドで段階移行する
- 移行完了後も一定期間はローカルと S3 の両読みに対応する
