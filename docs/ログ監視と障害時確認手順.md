# ログ監視と障害時確認手順

## 監視対象

- Web アプリケーションログ
- PostgreSQL ログ
- リバースプロキシログ
- OS のディスク使用率、メモリ使用率

## 平常時に見る項目

- `5xx` 発生件数
- ログイン失敗増加
- AI API エラー率
- 添付保存失敗件数
- ディスク使用率 80% 超過

## 障害時の確認順

1. `GET /health` と `GET /api/health` の応答確認
2. Web コンテナログの直近エラー確認
3. DB 接続エラーの有無確認
4. 直前デプロイ差分と `npm run risk:gate` 実行状況の確認
5. 添付だけ失敗している場合は保存先ディレクトリ権限と空き容量を確認
6. AI だけ失敗している場合は API キー設定と外部プロバイダー障害を確認

## 初動ルール

- 原因不明でも破壊的な復旧操作をしない
- まずログ保全、時刻記録、影響範囲の切り分けを行う
- データ破損が疑われる場合は書き込み系操作を止める
