# MT 管理者画面 UI 要件

## 目的

管理者が会社、ユーザー、案件の運用状態を UI 上で安全に確認、更新できるようにするための要件を整理する。

## 対象ユーザー

- システム管理者

## 最低限必要な画面

### 1. ユーザー一覧

- 会社で絞り込める
- メールアドレス、表示名、権限、所属会社を一覧表示する
- 有効 / 無効状態を確認できる
- ページネーションを持つ

### 2. ユーザー作成 / 編集

- 会社選択
- 表示名入力
- メールアドレス入力
- 権限選択
- パスワード設定 / 再設定
- 有効 / 無効切り替え

### 3. 会社一覧

- 会社名検索
- 類似名警告の確認
- 会社詳細への遷移

### 4. 会社詳細

- 案件一覧への導線
- 会社共通辞書への導線
- 所属ユーザー一覧への導線

## 操作要件

- 破壊的操作は確認ダイアログを必須とする
- 権限不足時は UI 上でも操作不可を明示する
- API エラー時はトーストではなく画面内にも文脈付きで表示する
- 類似名警告は保存阻止ではなく、警告表示で扱う

## 一覧 UI 要件

- 会社、ユーザー、案件の一覧は共通のページネーション UI を使う
- 検索条件変更時は 1 ページ目へ戻す
- 0 件時の空状態文言を用意する

## 今後の拡張余地

- 監査ログ参照
- 履歴参照
- 添付ファイル運用確認
- バックアップ状態確認
