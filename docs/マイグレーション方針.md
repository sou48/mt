# MT マイグレーション方針

## 目的

Prisma を前提に、ローカル開発、本番反映、将来のスキーマ更新を破綻なく進めるための方針を定める。

## 採用方針

- ORM / マイグレーション基盤は Prisma を採用する
- ローカルでは `prisma migrate dev` を使う
- 本番では `prisma migrate deploy` を使う
- `db push` は通常運用では使わない

## コマンド方針

### ローカル

```bash
npm run db:generate
npm run db:migrate:dev -- --name init
```

### 本番

```bash
npm run db:generate
npm run db:migrate:deploy
```

## 運用ルール

- スキーマ変更は必ず `prisma/schema.prisma` を更新する
- スキーマ変更後は migration ファイルを生成する
- migration を生成せずに本番へ持ち込まない
- 危険な一括再作成系コマンドは使わない
- DB を壊す操作は禁止する

## 初期実装の扱い

- 今回は Prisma スキーマと実行コマンドを整備する
- 初回 migration 作成は、Docker で PostgreSQL を起動した後に行う
- migration 名の初回候補は `init` とする

## 想定スクリプト

- `npm run db:generate`
- `npm run db:format`
- `npm run db:migrate:dev`
- `npm run db:migrate:deploy`

## 注意事項

- 論理削除を採用しているため、部分一意制約が必要な箇所はアプリケーション制御と併用する
- 未分類仮案件の「会社ごとに 1 件」はアプリケーション側の作成ロジックで担保する
